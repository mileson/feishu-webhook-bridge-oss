# 安全说明

## 请不要提交以下内容

- `.env`
- 任何 App Secret、API Key、Access Token
- 本地数据库文件
- 本地日志文件
- 私有服务地址或内网地址
- 带真实用户 ID、群聊 ID、会话 ID 的调试输出

## 发布前自查建议

在公开仓库前，建议至少检查：

```bash
rg -n "secret|token|api[_-]?key|app_secret|bearer|sk-|cr_" .
```

同时确认：

- `.gitignore` 已正确忽略敏感文件
- 提交历史中没有曾经提交过密钥
- 示例文档只使用占位符，不使用真实值

## 泄漏后的处理建议

如果你怀疑密钥已经泄漏：

1. 立即在对应平台吊销旧密钥
2. 生成新密钥
3. 更新本地 `.env`
4. 检查 Git 历史并清理泄漏记录
